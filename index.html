Overview

This website demonstrates what I have learned in Java since I have started programming in it. This website shows the homemade elements I have used various times in my projects. I wish to share this knowledge in the form of this website to help anyone who may need it. These classes and methods can become very useful if you use them correctly and in the right places.

  • In Java's Swing Library, I show the basics of using the java.awt abstract library, as well as its forerunning and non-abstract library, javax.swing.

  • In KeyChecker, I show the uses of my custom-made class KeyChecker, which extends java.awt.event.KeyAdapter. This class is used to detect key inputs from a java.awt.Component and to provide a basic method for checking if a key is pressed according to this Component.

  • In Hitboxes, I show how to detect collision with hitboxes and how to relate them to on-screen elements.

  • In Rotating Labels or RJLabels, I show how you can procedurally rotate a javax.swing.JLabel (or really any AWT Component), and how to override methods, add in your own code, and call the previous method as a result.

  • In Directional Movement, I show how you can use Math.sin() and Math.cos() to convert an integer value responsible for resembling a direction in degrees into two separate values, one for X movement, and the other for Y movement.

    • These are the pieces of programming knowledge I will share in this webpage. They mostly relate to each other, except for the Basic Network Communication and Launcher pages.     The Swing Library

    Java's Swing Library is a convenient class dedicated to providing windowing program abilities, as well as having components and containers within it/them. JFrame, JLabel, JPanel, and other core elements (those being JComponent objects and/or JContainer objects).

    Click here to learn more about the Swing library.

    KeyChecker

    This is a basic non-static class that listens to the key presses of a AWT Component, and keeps them in a java.util.ArrayList. You can then check the KeyChecker if a certain key is being pressed. When the key is released, the key is removed from the list. Each key is resembled with an integer value, according to the java.awt.event.KeyEvent virtual keyboard integer return values. (e.g. KeyEvent.VK_SPACE, KeyEvent.VK_F, etc.)

    Click here to learn more about the KeyChecker class.

    Hitboxes

    Here, I show — mostly with math — how to have two hitboxes detect collision with each other. I start with square hitboxes, but also how I suspect how circular hitboxes collide, though I have not tested this. Afterwards, I show how to correlate a hitbox with an on-screen entity.

    Click here to learn more about hitboxes and virtual collision.

    Rotating Labels (RJLabels)

    In this section, I show how to procedurally rotate JLabel objects by overriding their paintComponent() method and using Graphics2D to transform, rotate, and finalize the label's image. This is according to a dedicated rotation variable, measured in degrees. Since Graphics2D takes radians for its orientation-related methods, we also convert degrees into radians.

    Click here to learn more about rotating labels.

    Directional Movement

    In this section, I show how to have an on-screen element move in a particular direction measured in degrees. We use Math.sin() and Math.cos() to our advantage to do this. A live demonstration of how sine and cosine work can be found in this page. We convert a single degree-measured value into two separate values, one for X-position movement, and the other for Y-position movement. We can then use that value to adjust our on-screen element's position. I also show how to convert these values back into one, in which I demonstrate another situation where this is useful.

    Click here to learn more about orientation-based movement.
