RJLabels, the Rotatable Labels

Summary

The RJLabel class is a JLabel that has the easy ability to rotate, whether it is a piece of text or an image. Though that is the extent of this class, it can be useful in certain scenarios.

Usage

Here is how you use the class.

JFrame frame = new JFrame("Project");
// ...
RJLabel rjlabel = new RJLabel("Hello", 45);
frame.add(rjlabel);
frame.setVisible(true);

You use the RJLabel just like how you would typically use the JLabel. The only difference in this class is that there is a variable responsible for the orientation of the label, and that you can assign the default value of this variable in each applicable constructor, like seen above. The RJLabel here would be created with a 45o angle. After the label is created, you can also change its rotation with its rotateTo(int) method.

RJLabel rjlabel = new RJLabel("Hello", 0);
frame.add(rjlabel);
frame.setVisible(true);
sleep(5000);
rjlabel.rotateTo(22);

Or, better yet:

RJLabel rjlabel = new RJLabel("Hello", 0);
frame.add(rjlabel);
frame.setVisible(true);
int rotation = 0;
while (true) {
  rotation++;
  rotation %= 360;
  rjlabel.rotateTo(rotation);
  sleep(10);
}

The RJLabel Class

Here are the contents of the RJLabel class. There are six constructors, one method, and one overridden method (important).

package project.base;
import javax.swing.JLabel;
import java.awt.Icon;
import java.awt.Graphics;
import java.awt.Graphics2D;
public class RJLabel extends JLabel {
  private int rotation = 0;
  // The other six constructors are public, so I shouldn't need to override them
  /*
    Also, I have no choice but to put the super() method at the start as putting it
    anywhere else is an experimental part of Java at the time of writing.
  */
  public RJLabel(int rotation) {
    super();
    this.rotateTo(rotation);
  }
  public RJLabel(Icon image, int rotation) {
    super(image);
    this.rotateTo(rotation);
  }
  public RJLabel(Icon image, int horizontalAlignment) {
    super(image, horizontalAlignment);
    this.rotateTo(rotation);
  }
  public RJLabel(String text) {
    super(text);
    this.rotateTo(rotation);
  }
  public RJLabel(String text, Icon icon, int horizontalAlignment) {
    super(text, icon, horizontalAlignment);
    this.rotateTo(rotation);
  }
  public RJLabel(String text, int horizontalAlignment) {
    super(text, horizontalAlignment);
    this.rotateTo(rotation);
  }
  public rotateTo(int rotation) {
    /*
      It's important that we convert our degrees into radians, because degrees are much easier to use for
      us, but AWT needs radians for their methods.
    */
    this.rotation = Math.toRadians(rotation);
    this.repaint();
  }
  @Override
  public void paintComponent(Graphics g) {
    // We're hijacking this method and editing its graphics as it's being processed
    Graphics2D g2 = (Graphics2D) g;
    // This is to center the image when we rotate it, as its pivot point starts at the top-left
    int x = (int) this.getWidth() / 2;
    int y = (int) this.getHeight() / 2;
    // Center the image by offsetting its pivot point to the center
    g2.translate(x, y);
    // Actually rotate the image
    g2.rotate(this.rotation);
    // ...and set back the pivot point to what it was at before.
    g2.translate(-x, -y);
    // Use these edited graphics instead of the lame old ones
    super.paintComponent(g2);
    // Dispose of unused memory whenever possible. JGC doesn't catch everything
    g2.dispose();
  }
}

The comments should explain what is happening in the rotation process. We are overriding the paintComponent(Graphics) method and editing its graphics before it is actually processed by the original paintComponent(Graphics) method. In the long run, we are rotating the image according to our rotation variable before sending it off to the superclass's method.
